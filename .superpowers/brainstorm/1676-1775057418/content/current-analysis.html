잘 된 것

기술적 완성도

glassmorphism 카드, 스크롤 프로그레스, 모달 포커스 트랩, 접근성 — 코드 퀄리티는 높음

다크 테마 일관성

CSS 변수 체계, 섹션별 radial gradient 포인트 — 전체 색상 시스템이 잘 정리됨

개선 여지

1. "개발자 포트폴리오 보일러플레이트" 느낌

다크 배경 + 네온 그린 + 히어로에 코드 블록 — 이 조합은 GitHub에서 수백 개의 포트폴리오가 쓰고 있음. 구별되지 않음.

2. 히어로 섹션이 평평함

이름 + 코드블록 레이아웃은 정보는 전달하지만 기억에 남지 않음. 첫인상에서 "와" 하는 순간이 없음.

3. 섹션 간 리듬이 단조로움

About → Skills → Experience → Projects — 모두 비슷한 패딩, 비슷한 간격, 비슷한 무게감. 스크롤하면서 각 섹션이 구별되지 않음.

4. 타이포그래피 계층이 약함

폰트 크기 차이가 작고, 헤딩의 개성이 부족함. 텍스트가 "존재"하지만 "말을 걸지" 않음.

핵심 문제

사이트가 "프론트엔드 개발자가 만든 포트폴리오" 처럼 보이지, "정태윤이 만든 포트폴리오" 처럼 보이지 않음. 기술은 있는데 개성이 없음.
